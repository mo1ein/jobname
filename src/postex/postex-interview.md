### Technical Interview
- Could you introduce yourself?

- Can you tell me about the most recent Next.js project you worked on? How did you improve its accessibility?

- Have you worked with TanStack Query? Do you prefer using it over fetching data manually? Why?

- If you had to choose between the Fetch API and Axios, which one would you pick, and why?

- What are Promises in JavaScript, and why do we use them?

- Imagine you have four console.log() statements: two synchronous logs one after another, one inside a Promise, and one inside setTimeout(..., 0). In what order would they be printed, and why?

- Where would you store an authentication token: Local Storage or Cookies? Why?

- How have you implemented JWT-based authentication in your previous projects? Can you walk me through the authentication flow?

- What's the difference between throttling and debouncing? When would you use each one?

- If multiple API requests are sent one after another, how would you make sure an older request doesn't overwrite the result of a newer one if it finishes later?

- What's the most recent technical challenge you faced in your code, and how did you solve it?

- Suppose there's a bug in production that you can't reproduce locally. How would you investigate and fix it?

- How do you identify whether a React component is re-rendering too often, and how would you optimize it?

- Why might a useEffect end up in an infinite loop? What are the common causes?
